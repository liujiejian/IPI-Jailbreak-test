Agent Security: Risks from External Web Fetching
现代Agent具有调用工具的能力，其中 web_fetch, web_search 是一个重要的工具，用于从外部 web 获取数据；
此外，这些 agent 还可能采用周期性的任务，例如 weather daily, email notification/ auto reply, 等。
本研究旨在调查敌手能否通过 web_fetch (web_search 需要调研后再决定是否加入) 通过注入不安全的content，影响 Agent 的 memory.md，从而对未来执行任务产生安全影响。
Threat model
系统中涉及三个角色，user, agent/LLM (provider), API providers.
其中，用户是受害者, 敌手也就是API provider 目标是在用户使用 agent 执行自动化任务时，将特定的恶意内容写入到 agent 的 memory.md，从而在未来，用户/agent 产执行相关任务时产生安全风险。
Our work / contribution
我们提出了一个攻击路径，周期性任务+ web_fetch 实现对 memory.md 的注入，从而产生持久化的影响； 对此进行了安全建模
我们提出了新的攻击策略 （例如采用 status code，结合渐进式攻击策略）
我们构建了一个测试框架，实现对不同 agents 的相同场景/任务下的安全测试；
我们进行了测试，涵盖了 LLM (商业大模型，用户自己运行小模型，多个agent框架)

Openclaw
检测工具/架构
交互过程本质如下
User -- task --> Agent( LLM --> call tool --> tool response --> LLM) --> User interface
本质上， LLM 接收的包括 system_prompt, user_prompts, tools 等，
因此，我们理论上可以完美的模拟/组装这些内容发送给 LLM，并解析 LLM 的response 来进行操作/分析。
